## ‚öôÔ∏è Basic Setup

1. **Wrap** your application with the provider at the root:  
   ```tsx
   import { ToastProvider } from 'notify-toast';
   import 'notify-toast/styles.css';

   function App() {
     return (
       <ToastProvider
         defaultOptions={{
           position: 'top-right',
           duration: 5000,
           animation: 'slide',
         }}
         maxToasts={5}
         newestOnTop
       >
         <YourRoutesOrComponents />
       </ToastProvider>
     );
   }
   ```
2. **Invoke** toasts anywhere via the hook:  
   ```tsx
   import { useToast } from 'notify-toast';

   function Demo() {
     const { toast, success, error, info, warning } = useToast();
     return (
       <>
         <button onClick={() => toast('Hello World!')}>Default</button>
         <button onClick={() => success('It worked!')}>Success</button>
       </>
     );
   }
   ```
This pattern mirrors popular libraries like React‚ÄëToastify but brings a smaller footprint and TypeScript‚Äëfirst API. ÓàÄciteÓàÇturn1search0ÓàÇturn1search2ÓàÅ

## üìö Detailed Example

Below is the **exact** integration from your `App.tsx`‚Äîannotated to explain each feature:

```tsx
import './App.css';
import { useEffect } from 'react';
import AppRoutes from './routes/routes';

// 1) Bring in provider and hook
import { ToastProvider, useToast } from 'notify-toast';
import 'notify-toast/styles.css';

function ToastDemo() {
  const {
    toast, success, error, info, warning,
    loading, custom, update, dismissAll
  } = useToast();

  useEffect(() => {
    // 2) Show default toast
    toast('Default notification', { position: 'top-right' });

    // 3) Success after delay
    setTimeout(() => {
      success('Completed successfully!', {
        progressBar: true, duration: 4000
      });
    }, 1000);

    // 4) Error at bottom-left with bounce animation
    setTimeout(() => {
      error('Something went wrong.', {
        position: 'bottom-left', animation: 'bounce', duration: 5000
      });
    }, 2000);

    // 5) Rich Markdown content
    setTimeout(() => {
      info(
        '# Info\n\n- **Point A**\n- **Point B**',
        { richContent: true, duration: 6000 }
      );
    }, 3000);

    // 6) Warning with action button
    setTimeout(() => {
      warning('Delete this item?', {
        action: {
          label: 'Undo',
          onClick: () => info('Undo success!'),
          theme: 'danger'
        },
        duration: 7000
      });
    }, 4000);

    // 7) Loading ‚Üí update to success
    setTimeout(() => {
      const id = loading('Loading data...', { animation: 'fade' });
      setTimeout(() => {
        update(id, {
          type: 'success',
          content: 'Data loaded!',
          duration: 2000
        });
      }, 3000);
    }, 5000);

    // 8) Custom toast with sound
    setTimeout(() => {
      custom('üéâ Celebratory toast!', {
        sound: true, animation: 'flip', progressBar: true
      });
    }, 9000);

    // 9) Cleanup on unmount
    return () => dismissAll();
  }, []);

  return null;
}

function AppContent() {
  return (
    <>
      <AppRoutes />
      <ToastDemo />
    </>
  );
}

export default function App() {
  return (
    <div className="open-sans">
      <ToastProvider>
        <AppContent />
      </ToastProvider>
    </div>
  );
}
```

## ‚ôø Accessibility  
- **ARIA Live Regions**: Toasts render inside a `<div aria-live="assertive">` by default, ensuring screen readers announce them immediately ÓàÄciteÓàÇturn0search0ÓàÇturn0search4ÓàÅ.  
- **Alertdialog Role**: Each toast uses `role="alertdialog"` so users can navigate via F6/Shift+F6 landmarks ÓàÄciteÓàÇturn0search1ÓàÅ.  
- **Focus Management**: Dismissal restores focus to prior element or next toast, avoiding keyboard traps ÓàÄciteÓàÇturn0search3ÓàÅ.  
- **Pause/Resume**: Hover or focus events pause timers, letting assistive tech read long messages fully ÓàÄciteÓàÇturn0search6ÓàÅ.

## üöÄ Advanced Usage

### Updating & Queuing  
- Use `priority` to bump urgent toasts ahead of queued ones.  
- `group` + `dismissAll('groupName')` to clear related notifications.

### Server‚ÄëSide Rendering  
- ToastProvider guards against DOM usage on the server; no build‚Äëtime errors.  

### Performance  
- Minimal re‚Äërenders: only the toast list component updates.  
- Batch updates internally via React‚Äôs `useTransition`, avoiding jank.
